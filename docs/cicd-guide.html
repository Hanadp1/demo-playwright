üéØ Core Concepts

üîÑ

Continuous Integration (CI)

The practice of frequently merging code changes into a shared repository, where automated builds and tests verify each integration.

Key Practices:

  • Maintain a single source repository
  • Automate the build process
  • Make builds self-testing
  • Everyone commits to mainline daily
  • Fix broken builds immediately
üöÄ

Continuous Delivery (CD)

An extension of CI that automatically prepares code changes for release to production, ensuring software can be deployed at any time.

Key Practices:

  • Automate deployment pipeline
  • Keep production-like environments
  • Implement feature flags
  • Enable one-click deployments
  • Practice infrastructure as code

üìÑ GitHub Actions Workflow

Our pipeline is defined in .github/workflows/test.yml. Here's the complete configuration with explanations:

test.yml 
name: Playwright Tests CI

# Trigger conditions - when should the pipeline run?
on:
  push:
    branches: [ main, master ]
  pull_request:
    branches: [ main, master ]
  workflow_dispatch:              # Manual trigger from GitHub UI

# Required permissions for the workflow
permissions:
  contents: read
  checks: write
  pull-requests: write

jobs:
  test:
    runs-on: ubuntu-latest        # Virtual machine environment

    steps:
      # Step 1: Get the source code
      - name: Checkout code
        uses: actions/checkout@v4

      # Step 2: Install Java Development Kit
      - name: Set up JDK 11
        uses: actions/setup-java@v4
        with:
          java-version: '11'
          distribution: 'temurin'
          cache: maven            # Cache dependencies for faster builds

      # Step 3: Download project dependencies
      - name: Install dependencies
        run: mvn clean install -DskipTests

      # Step 4: Install browser for testing
      - name: Install Playwright browsers
        run: mvn exec:java -e -D exec.mainClass=com.microsoft.playwright.CLI \
             -D exec.args="install chromium"

      # Step 5: Execute test suite
      - name: Run tests
        run: mvn test -Dheadless=true

      # Step 6: Save test results as artifacts
      - name: Upload test reports
        uses: actions/upload-artifact@v4
        if: always()                  # Run even if tests fail
        with:
          name: test-reports
          path: target/surefire-reports/
          retention-days: 30

Workflow Steps Breakdown

1

Checkout Code

Clones the repository into the GitHub Actions runner, making source code available for subsequent steps.

actions/checkout@v4
2

Set up JDK 11

Installs Java Development Kit with Maven dependency caching for faster builds on subsequent runs.

actions/setup-java@v4
3

Install Dependencies

Downloads and installs all Maven dependencies defined in pom.xml, skipping tests during this phase.

mvn clean install
4

Install Playwright Browsers

Downloads Chromium browser binaries required for Playwright to execute browser automation tests.

playwright CLI
5

Run Tests

Executes the TestNG test suite in headless mode, validating application functionality without GUI.

mvn test
6

Upload Test Reports

Preserves test execution reports as downloadable artifacts for debugging and compliance purposes.

actions/upload-artifact@v4

üìã Best Practices

üîí Keep Pipelines Fast

Aim for builds under 10 minutes. Use caching, parallel jobs, and selective testing to reduce wait times.

üß™ Test at Multiple Levels

Implement unit, integration, and end-to-end tests. Follow the testing pyramid principle.

üîê Secure Your Pipeline

Use secrets management, scan for vulnerabilities, and follow least-privilege principles.

üìä Monitor & Measure

Track deployment frequency, lead time, failure rate, and recovery time (DORA metrics).

üè∑Ô∏è Version Everything

Use semantic versioning for releases and maintain infrastructure as code.

üìù Document Runbooks

Maintain clear documentation for pipeline maintenance and incident response.
